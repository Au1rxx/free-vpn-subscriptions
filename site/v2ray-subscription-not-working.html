V2Ray subscription not working: fix import, refresh, and connection problems

Use this page when a V2Ray subscription URL fails to import, stops refreshing, or loads nodes that still do not connect reliably.

https://raw.githubusercontent.com/Au1rxx/free-vpn-subscriptions/main/output/v2ray-base64.txt

Most common causes

  • The app expects another format such as Clash YAML or remote JSON.
  • The subscription was saved as a one-time import instead of a refreshable source.
  • The client cache is stale and needs a manual refresh.
  • The selected node is unhealthy and the status page was not checked first.

First checks

  • Confirm the app accepts Base64 subscription URLs.
  • Confirm the full V2Ray link is saved in the subscription source.
  • Run one manual refresh.
  • Test another region if the first node fails.

Step-by-step troubleshooting path

  1. Verify the app supports V2Ray subscription URLs rather than only single-node share links.
  2. Check that the saved source is the full Base64 URL above.
  3. Refresh the subscription manually and confirm the node list updates.
  4. Switch nodes or regions before changing app-wide settings.
  5. Use the status dashboard to separate node-side issues from client-side issues.

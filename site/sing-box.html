When the sing-box feed is the right choice

  • Your client accepts remote JSON configuration imports.
  • You want a sing-box-native profile path instead of a Clash YAML.
  • You are testing clients that parse structured JSON profiles more cleanly than Base64 links.

Update and verification path

  • Refresh cadence for the public JSON feed is every 6 hours.
  • The status page is refreshed hourly and should be checked before deep troubleshooting.
  • Release snapshots act as stable downloadable checkpoints.

How to import a sing-box remote profile

  1. Open your sing-box client and look for remote profile, import from URL, or remote configuration options.
  2. Create a new profile entry and paste the JSON URL shown above.
  3. Sync the profile once to load the current public node set.
  4. Refresh the profile manually so you can verify the client keeps the remote source.
  5. Select an outbound and test connectivity against the live status view.

Common sing-box import problems

  • If your client only accepts local files, download the release snapshot instead of forcing a URL flow that the app does not support.
  • If the profile imports but looks stale, trigger a manual sync and verify the source URL was saved correctly.
  • If outbounds load but traffic does not pass, compare with the published status page first.
  • If you are not sure whether to use sing-box or V2Ray, choose the format your client documents as the native remote profile path.

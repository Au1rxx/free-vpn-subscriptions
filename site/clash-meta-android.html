Why use the Clash YAML path

  • It matches the profile style most Clash-based Android users expect.
  • It keeps the import path consistent with Clash Verge Rev and FlClash.
  • It supports remote refresh better than manual one-off imports.

Good verification habits

  • Confirm the profile saved as a remote subscription.
  • Refresh once after import to verify the URL is reusable.
  • Check the live status dashboard before deep troubleshooting.

Steps

  1. Open Clash Meta for Android and go to profiles.
  2. Create a new profile from URL or add a remote subscription.
  3. Paste the Clash YAML link above.
  4. Import the profile and refresh it once manually.
  5. Select a node and test connectivity before changing app-wide settings.

Common issues

  • If the app imports a profile but connects poorly, switch nodes before rewriting DNS or routing settings.
  • If the remote profile does not refresh, re-open the profile and confirm the URL was saved as a subscription source.
  • If all nodes fail together, compare with the status page first.

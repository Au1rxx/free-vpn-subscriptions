Share the right public link

  • Use the raw `output/*` links for long-lived client auto-refresh.
  • Use `releases/latest/download/*` when you want a snapshot file for manual download.
  • Use the live status page or raw `status.json` when you need the freshest health signal before troubleshooting.
  • Open the verification page when you want the documented machine validation path and compatibility notes.

Primary public links

  • Clash: `clash.yaml`
  • sing-box: `singbox.json`
  • V2Ray: `v2ray-base64.txt`
  • Status: `status.json`

What each link is for

  • Clash for YAML and Mihomo-style clients
  • sing-box for remote JSON profile flows
  • V2Ray for Base64 subscription URL flows
  • Status for checking current node health before troubleshooting
